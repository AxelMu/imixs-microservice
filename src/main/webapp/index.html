Imixs-Microservice

Welcome

Imixs-Microservice provides a WebService Interface which is used to interact with the Imixs-Workflow-Engine. You can submit and request process instances of your business objects based on a JSON or XML structure.

Verifiy Model

First you need to check if a Model is still uploaded:
http://[YOURSERVER]/rest-service/model
If the XML is empty you need first to create a new model and updload it. You can upload the model with the following URL: 

http://[YOURSERVER]/rest-service/model

Worklist

You can request the task list for the current user with the following URL:
http://[YOURSERVER]/rest-service/workflow/worklist
The result will include all workitems for the current user.

Create a Workitem

To create a new process instance you can POST a JSON Object to the Imixs-Microservice in the following way 

URI= http://localhost/workflow/rest-service/workflow/workitem.json
METHOD=POST
MEDIA TYPE= application/json
To create a valid workitem the following attributes are mandatory:
  • $modelversion = the version of your model
  • $processid = the start process
  • $activityid = the activity to be processed
See the follwoing Example: 
 {"item":[
     {"name":"$modelversion","value":{"@type":"xs:string","$":"my-model-definition-0.0.2"}},
     {"name":"$processid","value":{"@type":"xs:int","$":"2000"}}, 
     {"name":"$activityid","value":{"@type":"xs:int","$":"1"}}, 
     {"name":"_subject","value":{"@type":"xs:string","$":"Some usefull data.."}}
   ]}
Once you created a new process instance based on a predefined model you got a lot of data form the workflow engine describing the state of your business object: 
{"item":[
   {"name":"$uniqueid","value":{"@type":"xs:string","$":"141cb98aecc-18544f1b"}},
   {"name":"$modelversion","value":{"@type":"xs:string","$":"bestandspflege-de-0.0.2"}},
   {"name":"$processid","value":{"@type":"xs:int","$":"2000"}},
   {"name":"namcreator","value":{"@type":"xs:string","$":"admin"}}, 
   {"name":"namcurrenteditor","value":{"@type":"xs:string","$":"admin"}}, 
   {"name":"namowner","value":{"@type":"xs:string","$":"admin"}}, 
   {"name":"$isauthor","value":{"@type":"xs:boolean","$":"true"}},
   {"name":"_subject","value":{"@type":"xs:string","$":"JUnit Test-6476"}}, 
   {"name":"txtworkflowstatus","value":{"@type":"xs:string","$":"Vorlauf"}}, 
   {"name":"txtworkflowresultmessage","value":{"@type":"xs:string","$":""}}
  ]}
This includes the attribute '$uniqueid' wich is used ot identify the process instance. Also workflow information like the current status or the owner is returend by the service.

There are serveral Resouce URIs to request the state of a process instance. Using the $uniqueid returned by the POST method you can request a single workitem: 

URI= http://localhost/workflow/rest-service/workflow/workitem/[UNIQUEID].json
METHOD=GET
MEDIA TYPE= application/json
or you can also request a list of all workitems for the curren user: 
URI= http://localhost/workflow/rest-service/workflow/workitem/worklist.json
METHOD=GET
MEDIA TYPE= application/json
